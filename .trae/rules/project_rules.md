# Git Commit 规范

## 提交信息格式

```
<类别>(范围): 英文总结

1. 改动点1
2. 改动点2
3. 改动点3
```

## 类别标识

- **feat** - 新功能
- **fix** - 修复 bug
- **build** - 构建相关
- **ci** - CI 配置
- **docs** - 文档更新
- **style** - 代码格式（不影响功能）
- **refactor** - 重构
- **perf** - 性能优化
- **test** - 测试相关
- **chore** - 其他杂项

## 示例

```
feat(auth): Add user authentication feature

1. 实现登录接口
2. 添加 JWT token 验证
3. 修复密码加密问题
```
